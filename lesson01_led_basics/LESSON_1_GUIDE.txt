Lesson 1 — Lights and buzzer (Module 1)
=======================================

What this is (very short)
-------------------------
**LED:** still simple — pin HIGH or LOW, light on or off.

**Buzzer:** this kit uses a **passive** buzzer. A steady HIGH does **not** make a proper tone — the disk needs to **vibrate many times per second**. The lesson does that with **PWM** (pulse-width modulation): the pin toggles at a **frequency** (Hz), which you hear as pitch. Longer `duration_ms` = longer beep.


Words worth knowing
---------------------
GPIO / HIGH / LOW / BCM / output — same as before for the **LED** only.

Passive buzzer — Needs an **alternating** drive (here: PWM), not “DC on forever.”

PWM — Pin switches on/off very fast; **frequency** (e.g. 2500 Hz) sets pitch; **duty** here is 50% for a clear square-ish tone.

Hz — Cycles per second; higher number often sounds “sharper” for this hardware.


What you must fill in
---------------------
1) LED_PIN — BCM for red LED (kit: **26** unless your sheet differs).
2) ON_STATE_KEYWORD — `"HIGH"` or `"LOW"` for the LED.
3) BUZZER_PIN — BCM for buzzer (example **22** — follow your kit).
4) BUZZER_FREQ_HZ — optional; default **2500**; try **2000–4000** if tone is weak or harsh.
5) student_flow() — LED block first, then `setup_buzzer`, then three `passive_tone` calls (two short, one long) with `wait_ms` gaps between them.


Every function in the challenge code (what it does)
-----------------------------------------------------
student_flow()
  Your edit zone: LED sequence, then buzzer pattern.

set_on_state / setup_led / led_on / led_off / wait_seconds
  Same as before — only for the red LED.

setup_buzzer(pin)
  Prepares the buzzer pin as output and leaves line LOW (quiet) before tones.

passive_tone(duration_ms, frequency_hz)
  Starts PWM on BUZZER_PIN at `frequency_hz`, runs for `duration_ms`, then stops PWM so the buzzer is silent again.

wait_ms(milliseconds)
  Short pause between beeps (silence gap).

_stop_buzzer_pwm()
  Internal — stops PWM; you do not call it; `passive_tone` and `safe_cleanup` use it.

safe_cleanup()
  Stops buzzer PWM, then GPIO.cleanup().

run_with_guard() / main()
  Errors + always cleanup.


Hints (small)
-------------
• LED block first (it calls `GPIO.setmode` the first time); then `setup_buzzer` (safe if mode already set).
• Pattern: `passive_tone(100, BUZZER_FREQ_HZ)` style for short, larger ms for long; `wait_ms(80)` between beeps is a starting point.
• If buzzer is very quiet, try another BUZZER_FREQ_HZ in the 2–4 kHz range.
• If you hear clicks or weird noise, check wiring and that BUZZER_PIN matches the passive buzzer signal pin.


How to run
----------
  cd .../lesson01_led_basics
  python3 challenge.py


Success check
-------------
LED on ~3 s, off. Then two short tones, one longer tone, silence. `Cleanup complete.` after run or Ctrl+C.
