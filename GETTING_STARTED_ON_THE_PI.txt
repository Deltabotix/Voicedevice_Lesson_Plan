Getting started on the Raspberry Pi — lessons folder
======================================================

This kit keeps lesson code in the  lessons  folder. Use the commands below
in a terminal (black window on the Pi desktop, or SSH).


Where am I?
-----------
After login, open a terminal and go to the lessons folder:

  cd ~/lessons

If that fails, try:

  cd ~/Voicedevice/lessons
  cd /opt/voicedevice/lessons

Check you are in the right place:

  ls

You should see folders like  lesson_01_red_led ,  run ,  reset , and shared
files ( groq_env.py ,  tm_audio_runtime.py ). Python packages are NOT in this
folder — they live in  /opt/voicedevice/voicekit_v2/venv  (same as voicekit).

Shared files (same folder as  lesson_1 , not inside each lesson):
  groq_env.py           — lessons 8 and 9 (Groq key from kit Wi‑Fi setup)
  tm_audio_runtime.py   — lessons 8, 9, and 10 (Teachable Machine audio)
  labels.txt            — lessons 8–10 (from your TM model export)

If lesson 6 says  No module named groq_env , copy  groq_env.py  into  ~/lessons .


Running a lesson (recommended)
------------------------------
Lessons use the kit Python at  /opt/voicedevice/voicekit_v2/venv  — the same
environment as voicekit_v2. You do not create venv or run pip install.

Use the  run  script — it stops the Wi-Fi captive portal while you work,
then turns the portal back on when you finish.

  cd ~/lessons
  ./run 1
  ./run.sh 1                 # same as ./run

Shortcuts:

  ./run 6                  → lesson 6
  ./run lesson_06_led_prompt
  ./run lesson_10/challenge.py

When the lesson is running, press  Ctrl+C  to stop. The captive portal
service starts again automatically.


Getting new lessons (Git)
-------------------------
Lesson code updates from GitHub. The voicekit venv is unchanged.

  cd ~/lessons
  ./update.sh
  # or: git pull


Why ./run stops the captive portal
----------------------------------
The kit runs  captiveportal.service  so you can connect Wi-Fi and enter
your Groq key. While a lesson uses the microphone, speaker, or GPIO, the
portal is paused so nothing fights for those resources.  ./run  handles
that for you.


Editing code with nano
----------------------
  nano lesson_1/challenge.py

Use the arrow keys to move. Edit the lines marked  TODO  or  student_flow .

Save and exit:
  Ctrl+O   then Enter   (write file)
  Ctrl+X                (exit nano)

Cancel without saving:
  Ctrl+X   then  N  when asked to save


Listing and moving between folders
----------------------------------
  ls              list files in this folder
  ls lesson*      list all lesson folders
  cd lesson_1
  cd ..           go up one folder
  pwd             print current folder path


Stopping a running program
--------------------------
  Ctrl+C          stop the program in the foreground (lessons, tests)
  Ctrl+Z          suspend (advanced — avoid unless you know job control)

If you used  ./run , Ctrl+C also restarts the captive portal.


Copying a file from your laptop (optional)
-----------------------------------------
From your computer (not on the Pi), if you know the Pi IP address:

  scp myfile.wav voicedevice@192.168.x.x:~/lessons/lesson08_barnyard_tm/

Replace the IP and path with yours.


Groq API key (lessons 8 and 9)
------------------------------
Your key was saved when you set up Wi‑Fi on the kit (captive portal) into
/home/voicedevice/.env — you do not need to create a separate file for lessons.

If a lesson says the key is missing or Groq returns an auth error, finish Wi‑Fi
setup again or ask your teacher.

Then run the lesson with  ./run 8  or  ./run 9


Reset a lesson (Git)
--------------------
  cd ~/lessons
  ./reset 1                 # one lesson (my_program.py)
  ./reset --all               # all tracked files

Or:  git checkout -- lesson_01_red_led/my_program.py


If ./run asks for a password twice
----------------------------------
The kit pauses  captiveportal.service  while a lesson runs, then starts it again.
That is **two** systemctl actions (stop + start).

Production images should install passwordless sudo for that service only:

  sudo cp ~/lessons/voicedevice-captiveportal.sudoers /etc/sudoers.d/
  sudo chmod 440 /etc/sudoers.d/voicedevice-captiveportal.sudoers
  sudo visudo -c

After that,  ./run  should not ask for a password.

If you see "Choose identity 1 or 2", Polkit found two login accounts for the
same user — pick 1 (tempadmin) or ask your teacher. Installing the sudoers file
above avoids these prompts.

To skip captive portal control entirely (testing):

  LESSONS_CAPTIVE=skip ./run 1

To allow interactive sudo (old behaviour):

  LESSONS_CAPTIVE=prompt ./run 1


If ./run says permission denied
-------------------------------
  chmod +x ~/lessons/run ~/lessons/reset

If systemctl needs a password every time, install voicedevice-captiveportal.sudoers
on the image (see section above).


Quick reference
---------------
  cd ~/lessons
  ls
  nano lessonNN_.../challenge.py
  ./run lessonNN_.../challenge.py
  Ctrl+C   stop lesson + restart captive portal
